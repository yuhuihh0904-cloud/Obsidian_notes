## Example Exam Lookup

### If the exam question says:

> Explain the CUDA execution model.

You should check:

```text
Lecture 2 p6-p9
```

Answer idea:

```text
A CUDA program follows a heterogeneous execution model. The host code runs on the CPU and handles serial control, memory allocation, data transfer, and kernel launch. The device code runs on the GPU as kernels. A kernel is executed by a grid of thread blocks, and each block contains many threads. CUDA follows the SPMD model, where all threads execute the same program but process different data.
```

中文理解：

```text
CUDA 程序采用异构执行模型。Host code 在 CPU 上运行，负责串行控制、内存分配、数据传输和 kernel 启动；Device code 在 GPU 上运行，以 kernel 的形式由大量线程并行执行。Kernel 由 grid 执行，grid 包含多个 thread block，每个 block 包含多个 thread。CUDA 使用 SPMD 模型，即所有线程执行同一段程序，但处理不同数据。
```

### If the exam question says:

> Why is boundary checking necessary in CUDA kernels?

You should check:

```text
Lecture 2 p24-p28
```

Answer idea:

```text
Boundary checking is necessary because the number of launched threads is usually rounded up to match the block size. Therefore, some threads may have indices greater than or equal to the actual data size. Without boundary checking, these threads may access memory out of range and cause incorrect behavior.
```

中文理解：

```text
边界检查是必要的，因为 CUDA 启动线程时通常会按照 block size 向上取整，所以总线程数可能大于实际数据规模。某些线程的编号可能大于等于 N，如果不做边界检查，就可能访问数组越界，导致错误。
```
