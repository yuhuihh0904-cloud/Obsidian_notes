## Lecture 4: Memory Model

| Module / Topic | English Outline | 中文说明 | PPT Pages | Priority |
|---|---|---|---|---|
| Von-Neumann Model and Instruction Processing | Von-Neumann model, instruction fetch/decode/execute, operate instruction, load instruction. | 冯诺依曼模型与指令执行流程：计算机由 memory、input/output、control unit、processing unit 构成；指令需要经历 fetch、decode、execute、memory 等阶段。ADD 属于运算指令，LDR 属于访存指令。 | Lec4 p3-p6 | 了解 |
| Registers vs Memory | Registers are fast but few; memory is large but slow. | 寄存器速度快但容量小，内存容量大但访问慢。这是后面 CUDA 内存优化的背景。 | Lec4 p7 | 一般 |
| CUDA Memory Hierarchy | CUDA threads can access registers, shared memory, global memory, and constant memory. | CUDA 内存层次：register 是线程私有最快存储；shared memory 是 block 内共享的高速存储；global memory 所有线程可见但慢；constant memory 是只读且带缓存的 device memory。 | Lec4 p8-p9 | 重点 |
| Matrix Multiplication Basics | Matrix multiplication computes each output element as the dot product of one row of M and one column of N. | 矩阵乘法基础：每个 `P[i][j]` 等于 M 的第 i 行和 N 的第 j 列对应元素乘积之和。CPU 版本是三重循环，CUDA 中可以让每个线程计算一个输出元素。 | Lec4 p10-p13 | 重点 |
| Floating-Point Non-Associativity | Floating-point addition is not strictly associative because of rounding errors. | 浮点数不严格满足结合律，因为有限精度会导致舍入误差。并行 reduction 或改变加法顺序时，浮点结果可能和串行结果略有不同。 | Lec4 p14-p15 | 一般偏重点 |
| 2D Grid and Block for Matrix Multiplication | Use a 2D grid and 2D blocks to map threads to output matrix elements. | 用二维 grid 和二维 block 计算矩阵乘法。每个 thread 通过 `Row` 和 `Col` 定位自己负责的输出元素。 | Lec4 p16-p20 | 重点 |
| MatrixMulKernel | Each thread computes one `P[Row][Col]` using a loop over k. | 矩阵乘法 kernel：线程先计算 `Row` 和 `Col`，再循环 k，累加 `M[Row][k] * N[k][Col]`，最后写回 `P[Row][Col]`。 | Lec4 p21-p25 | 必看 |
| Global Memory Bottleneck | Basic matrix multiplication is limited by repeated global memory accesses. | 基础矩阵乘法性能瓶颈：每个线程反复从 global memory 读取 M 和 N，global memory 延迟高、带宽有限，所以程序可能是 memory-bound。后续 shared memory tiling 就是为了解决这个问题。 | Lec4 p26-p30 | 必看 |
| Problem Solving | Count blocks for 2D matrices; choose shared memory for communication within a block; understand nondeterministic writes across blocks. | 习题部分：会考二维矩阵需要多少 block、同一 block 内线程共享数据应使用 shared memory、多个 block 写同一地址时结果不确定。 | Lec4 p32-p34 | 必看 |
