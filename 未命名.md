| What is data parallelism? | Data parallelism means applying the same operation to many independent data elements. | 数据并行是指对大量相互独立的数据元素执行相同操作。 | Lec2 p4-p5 |
| Why is image grayscale conversion data parallel? | Each output pixel depends only on its corresponding input pixel, so pixels can be processed independently. | 图像灰度转换中，每个输出像素只依赖对应输入像素，因此不同像素可以独立处理。 | Lec2 p4-p5 |

# Lecture 3：Kernel-Based Data Parallel Execution Model

| Question / Keyword | English Answer Skeleton | 中文翻译 | PPT Page |
|---|---|---|---|
| Review vector addition | The vector addition kernel maps each thread to one output element using `blockIdx`, `blockDim`, and `threadIdx`. | 向量加法 kernel 通过 `blockIdx`、`blockDim`、`threadIdx` 把每个线程映射到一个输出元素。 | Lec3 p3-p4 |
| Total number of launched threads | Total launched threads equal `gridDim.x * blockDim.x` in a 1D kernel. | 一维 kernel 中，总启动线程数等于 `gridDim.x * blockDim.x`。 | Lec3 p3-p4 |
| Coarser-grain thread assignment | A thread can process more than one data element, reducing the number of blocks and increasing work per thread. | 一个线程可以处理多个数据元素，从而减少 block 数量并增加每个线程的工作量。 | Lec3 p5-p6 |
| Two elements per thread | For two elements per thread, the block starting index becomes `blockIdx.x * (2 * blockDim.x)`. | 每个线程处理两个元素时，block 起始下标变为 `blockIdx.x * (2 * blockDim.x)`。 | Lec3 p5-p6 |
| 2D grid for image processing | Images are naturally two-dimensional, so CUDA often uses 2D grids and 2D blocks for image processing. | 图像天然是二维数据，因此 CUDA 通常用二维 grid 和二维 block 处理图像。 | Lec3 p7-p10 |
| 2D thread mapping | `Col = threadIdx.x + blockIdx.x * blockDim.x`, `Row = threadIdx.y + blockIdx.y * blockDim.y`. | 二维线程映射公式为：`Col = threadIdx.x + blockIdx.x * blockDim.x`，`Row = threadIdx.y + blockIdx.y * blockDim.y`。 | Lec3 p11 |
| Row-major layout | In C/C++, a 2D array is stored in row-major order, so the linear index is `Row * Width + Col`. | C/C++ 中二维数组按行优先存储，一维下标为 `Row * Width + Col`。 | Lec3 p12 |
| RGB to grayscale offset | A grayscale image uses one value per pixel, while an RGB image uses three values per pixel, so `rgbOffset = 3 * greyOffset`. | 灰度图每个像素一个值，RGB 图像每个像素三个通道，所以 `rgbOffset = 3 * greyOffset`。 | Lec3 p11-p13 |
| Why boundary check in 2D kernels? | Boundary checks such as `Col < width && Row < height` prevent threads outside the valid image region from accessing invalid memory. | 二维 kernel 中的 `Col < width && Row < height` 用于防止超出图像范围的线程访问非法内存。 | Lec3 p10-p13 |
| Image blur | Each output pixel is the average of nearby input pixels within a blur window. | 图像模糊中，每个输出像素是其周围邻域像素的平均值。 | Lec3 p14-p16 |
| Boundary handling in blur | Pixels near image edges need boundary checks because their neighborhoods may extend outside the image. | 图像边缘像素的邻域可能超出图像范围，因此需要边界检查。 | Lec3 p16-p18 |
| CUDA thread blocks | A thread block contains multiple threads executing the same kernel; threads in a block may cooperate. | 一个 thread block 包含多个执行同一 kernel 的线程，block 内线程可以协作。 | Lec3 p19 |
| Compute capability | Compute capability describes the hardware features and resource limits of a CUDA GPU. | Compute capability 表示 CUDA GPU 的硬件特性和资源限制。 | Lec3 p20-p21 |
| Streaming Multiprocessor, SM | Thread blocks are assigned to Streaming Multiprocessors in block granularity. | Thread block 会以 block 为粒度分配到 Streaming Multiprocessor 上执行。 | Lec3 p22 |
| Warp | A warp is a group of threads scheduled together by an SM. It is typically 32 threads on NVIDIA GPUs. | Warp 是 SM 一起调度的一组线程，在 NVIDIA GPU 上通常是 32 个线程。 | Lec3 p23-p24 |
| Latency hiding | GPUs hide memory latency by switching to other ready warps while one warp waits for memory. | 当某个 warp 等待内存时，GPU 会切换执行其他 ready warp，从而隐藏内存延迟。 | Lec3 p23-p24 |
| Branch divergence | Branch divergence occurs when threads in the same warp take different control-flow paths. | Branch divergence 指同一个 warp 内线程走不同控制流路径。 | Lec3 p25-p27 |
| Why branch divergence hurts performance | Divergent branches are serialized within a warp, reducing parallel efficiency. | 同一个 warp 内不同分支会被串行执行，从而降低并行效率。 | Lec3 p25-p27 |
| Block granularity and occupancy | Block size affects how many blocks and threads can reside on an SM, which affects occupancy. | Block 大小会影响一个 SM 上可驻留的 block 和 thread 数量，从而影响 occupancy。 | Lec3 p28 |
| Problem solving: block size | To choose a block size, consider maximum threads per SM, maximum blocks per SM, occupancy, and resource usage. | 选择 block size 时要考虑每个 SM 最大线程数、最大 block 数、occupancy 和资源使用。 | Lec3 p30-p32 |

| Matrix multiplication formula | Each output element is computed as `P[i][j] = sum_k M[i][k] * N[k][j]`. | 矩阵乘法中，每个输出元素为 `P[i][j] = sum_k M[i][k] * N[k][j]`。 | Lec4 p10-p12 |
| Parallelizing matrix multiplication | Each CUDA thread can compute one element of the output matrix. | CUDA 中可以让每个线程计算输出矩阵的一个元素。 | Lec4 p13-p17 |

# Lecture 5：Locality and Tiled Matrix Multiplication

| Question / Keyword | English Answer Skeleton | 中文翻译 | PPT Page |
|---|---|---|---|
| Why is basic matrix multiplication slow? | Basic matrix multiplication performs many global memory accesses, and global memory bandwidth limits performance. | 基础矩阵乘法慢是因为它进行了大量 global memory 访问，性能受内存带宽限制。 | Lec5 p4-p7 |
| 4B per FLOP | In basic matrix multiplication, each multiply-add reads two floats, or 8 bytes, for 2 FLOPs, giving 4B per FLOP. | 基础矩阵乘法中，每次乘加读取两个 float，即 8 字节，完成 2 个 FLOP，因此约为 4B/FLOP。 | Lec5 p5 |
| Memory bandwidth limit | If a GPU has 150 GB/s bandwidth and requires 4B/FLOP, performance is limited to 37.5 GFLOP/s. | 如果 GPU 带宽为 150GB/s，且每 FLOP 需要 4B，则性能上限约为 37.5 GFLOP/s。 | Lec5 p6 |
| Solution: reuse memory accesses | To approach peak compute performance, global memory accesses must be reduced through data reuse. | 为接近峰值计算性能，必须通过数据复用减少 global memory 访问。 | Lec5 p7 |
| Tiled matrix multiplication | Tiled matrix multiplication breaks matrices into tiles, loads tiles into shared memory, and reuses them. | 分块矩阵乘法把矩阵切成 tile，加载到 shared memory 中并重复使用。 | Lec5 p8-p10 |
| Why use shared memory? | Shared memory is much faster than global memory and can be shared by threads in the same block. | Shared memory 比 global memory 快得多，并且可以被同一个 block 内线程共享。 | Lec5 p9-p14 |
| Shared memory declaration | `__shared__ float subTileM[TILE_WIDTH][TILE_WIDTH]` declares a shared tile for block-level reuse. | `__shared__ float subTileM[TILE_WIDTH][TILE_WIDTH]` 声明了一个供 block 内线程复用的 shared tile。 | Lec5 p11 |
| Tiling phases | The kernel is divided into phases; each phase loads one tile of M and one tile of N and computes a partial sum. | Tiled kernel 被分成多个 phase，每个 phase 加载一个 M tile 和一个 N tile，并计算部分和。 | Lec5 p15-p16 |
| Loading a tile | All threads in a block participate in loading M and N tiles into shared memory. | 一个 block 内所有线程共同参与，把 M 和 N 的 tile 加载到 shared memory。 | Lec5 p16-p20 |
| Coalesced tile loading | Tile loading should assign elements to threads so that accesses within a warp are coalesced. | 加载 tile 时应让同一 warp 内线程访问连续地址，从而实现 coalesced access。 | Lec5 p16-p17 |
| `__syncthreads()` after loading | Synchronization ensures all tile elements are loaded before computation begins. | 加载后同步可以保证所有 tile 元素都加载完毕后再开始计算。 | Lec5 p24-p30 |
| `__syncthreads()` after computation | Synchronization ensures all threads finish using the current tile before it is overwritten by the next phase. | 计算后同步可以保证所有线程用完当前 tile 后，再进入下一轮并覆盖 shared memory。 | Lec5 p24-p30 |
| Tiled kernel benefit | Tiling converts many global memory accesses into faster shared memory accesses. | Tiling 把大量 global memory 访问转换成更快的 shared memory 访问。 | Lec5 p34-p35 |
| Why not put whole matrix in shared memory? | Shared memory is fast but small, so large matrices must be processed tile by tile. | Shared memory 虽快但容量小，大矩阵无法整体放入，只能按 tile 处理。 | Lec5 p9-p10 |

# Lecture 6：More on Tiling

| Question / Keyword | English Answer Skeleton | 中文翻译 | PPT Page |
|---|---|---|---|
| What assumption did Lecture 5 make? | The tiled kernel assumed that matrix dimensions are multiples of `TILE_WIDTH`. | Lecture 5 的 tiled kernel 假设矩阵维度是 `TILE_WIDTH` 的整数倍。 | Lec6 p3-p4 |
| Why is padding not ideal? | Padding avoids boundary issues but requires reformatting data and increases transfer time. | Padding 可以避免边界问题，但需要重新格式化数据并增加数据传输时间。 | Lec6 p3 |
| Boundary condition in tiled matrix multiplication | When matrix size is not divisible by tile size, some tile elements fall outside valid matrix bounds. | 当矩阵大小不能被 tile size 整除时，部分 tile 元素会超出矩阵有效范围。 | Lec6 p5-p11 |
| Undefined tile values | Out-of-bound tile positions do not correspond to valid matrix elements, so their values are undefined unless handled. | 越界 tile 位置不对应有效矩阵元素，如果不处理，其值是未定义的。 | Lec6 p7-p10 |
| How to handle out-of-bound M tile values? | Load valid M elements normally; set out-of-bound shared memory positions to zero. | 对 M tile，合法位置加载真实值，越界位置在 shared memory 中设为 0。 | Lec6 p5-p11 |
| How to handle out-of-bound N tile values? | Load valid N elements normally; set out-of-bound shared memory positions to zero. | 对 N tile，合法位置加载真实值，越界位置在 shared memory 中设为 0。 | Lec6 p5-p11 |
| Why set invalid tile values to zero? | Zero values do not affect the dot product because multiplying by zero contributes nothing. | 无效 tile 值设为 0 不会影响点积结果，因为乘以 0 不贡献结果。 | Lec6 p5-p11 |
| How to handle out-of-bound P writes? | Write `P[Row][Col]` only if `Row < Width` and `Col < Width`. | 只有当 `Row < Width` 且 `Col < Width` 时才写回 `P[Row][Col]`。 | Lec6 p8-p11 |
| Number of tile phases | Use ceiling division: `(Width + TILE_WIDTH - 1) / TILE_WIDTH`. | tile phase 数量应使用向上取整：`(Width + TILE_WIDTH - 1) / TILE_WIDTH`。 | Lec6 p3-p4 |
| Why not return early before `__syncthreads()`? | All threads in a block must reach `__syncthreads()`; otherwise, the block may deadlock. | 一个 block 内所有线程都必须执行到 `__syncthreads()`，否则可能死锁。 | Lec6 p4-p11 |
| Boundary-safe tiled kernel | A correct kernel checks bounds when loading M, loading N, and writing P. | 正确的边界版 tiled kernel 需要在加载 M、加载 N、写回 P 三处做边界判断。 | Lec6 p4-p11 |

# Lecture 7：DRAM Bandwidth

| Question / Keyword | English Answer Skeleton | 中文翻译 | PPT Page |
|---|---|---|---|
| Ideal vs real DRAM bandwidth | Theoretical DRAM bandwidth may be high, but real programs may not fully use it due to inefficient access patterns. | 理论 DRAM 带宽可能很高，但实际程序由于访问模式不佳，未必能充分利用。 | Lec7 p3 |
| What is RAM? | RAM is memory where any address can be read or written in roughly the same time. | RAM 是随机访问内存，任意地址读写时间大致相同。 | Lec7 p4 |
| What is DRAM? | DRAM stores bits as charge on capacitors and must be periodically refreshed. | DRAM 用电容上的电荷存储 bit，并且需要周期性刷新。 | Lec7 p4 |
| Why is DRAM dynamic? | Stored charge leaks over time, so bits must be rewritten periodically. | 电容电荷会随时间泄漏，因此 bit 必须周期性重写，所以称为 dynamic。 | Lec7 p4 |
| Bit line | Many DRAM cells share a bit line, and select lines choose which cell connects to it. | 多个 DRAM cell 共享一条 bit line，通过 select line 选择哪个 cell 连接到 bit line。 | Lec7 p5 |
| Why is DRAM slow but dense? | DRAM uses only one transistor per bit, giving high density, but bit lines have large capacitance and require sensing. | DRAM 每 bit 只需一个晶体管，所以密度高，但 bit line 电容大，需要 sense amplifier，因此速度慢。 | Lec7 p6 |
| DRAM bank organization | A DRAM bank includes a core array, row decoder, sense amplifiers, column latches, mux, and pin interface. | 一个 DRAM bank 包括 core array、row decoder、sense amplifiers、column latches、mux 和 pin interface。 | Lec7 p7 |
| DRAM interface clock | DRAM cells are not clocked, but DRAM interfaces are clocked. | DRAM 存储单元本身不按时钟工作，但 DRAM 接口按时钟传输。 | Lec7 p8 |

## 2. Boundary checking

```text
Why is boundary checking necessary?
```

```text
CUDA 启动线程时通常按照 block size 向上取整，因此总线程数可能大于实际数据规模。超出数据范围的线程如果继续访问数组，会导致越界访问和错误结果。因此 kernel 中常使用 `if (i < N)` 或 `if (Row < height && Col < width)` 进行边界检查，保证只有有效线程访问有效数据。
```

## 3. Shared memory tiling

English trigger:

```text
Why does tiling improve performance?
```

中文作答模板：

```text
Tiling 的核心思想是把 global memory 中会被重复访问的数据块加载到 shared memory 中。由于 shared memory 比 global memory 快得多，并且同一个 block 内线程可以共享它，因此多个线程可以重复使用同一份数据，减少 global memory 访问次数。对于矩阵乘法和卷积这类访存密集型程序，tiling 可以显著降低内存带宽压力，提高性能。
```

## 4. Branch divergence

```text
What is branch divergence?
```


## 9. Histogram privatization

English trigger:

```text
How does privatization improve histogram performance?
```

中文作答模板：

```text
基础 histogram 中，所有线程直接对 global histogram 的 bin 执行 atomicAdd，若多个线程更新同一 bin，会产生严重 contention。Privatization 为每个 block 创建一个 private histogram，通常放在 shared memory 中。线程先更新本 block 的 private histogram，最后再把各 block 的结果合并到 global histogram。这样可以减少 global atomic 次数，并把大部分更新转移到更快的 shared memory 中，从而提高性能。
```
