# 如何赋予 LLM 规划能力？

## 一、面试标准回答

如果面试官问：

> 你是怎么给 LLM 加规划能力的？CoT、ToT、GoT 和 Plan-and-Execute 分别是什么？

可以这样回答：

我理解给 LLM 加规划能力，本质上是为了让模型不要直接“一口气生成答案”，而是把隐式推理过程显式化，让复杂任务可以被拆解、检查和纠偏。

最基础的方法是 **CoT，Chain of Thought**。  
它的核心是让模型把推理步骤写出来，比如在 prompt 里加入“请一步步思考”。这样模型前面生成的推理步骤会进入上下文，影响后续 token 的生成，相当于让模型在“纸上演算”，减少跳步出错。但 CoT 的问题是它只有一条推理链，如果一开始方向错了，后面会一路错下去。

为了解决 CoT 单路径的问题，可以用 **ToT，Tree of Thoughts**。  
ToT 不再只生成一条推理链，而是同时生成多条候选思路，然后对这些思路进行评估、打分和剪枝，只保留更有希望的路径继续展开。它的核心是：
