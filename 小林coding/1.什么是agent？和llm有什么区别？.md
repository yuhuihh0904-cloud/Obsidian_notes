# 面试标准答案

## 问题：什么是 Agent？与大模型有什么本质不同？

可以这样回答：

> 我理解的 **AI Agent** 是一个能够围绕目标**自主规划、调用工具、多步执行**，并根据环境反馈不断调整的 AI 系统。
>
> 普通大模型更多是一个**被动的文本生成器**，用户问一句，它回答一句，本质上是一次性的输入输出。而 Agent 不只是回答问题，它更像一个**任务执行者**。用户给它一个目标之后，它可以自己拆解任务、制定计划、选择合适的工具去执行，并根据工具返回结果继续判断下一步怎么做，直到完成目标。
>
> 所以 Agent 和普通大模型的本质区别主要有三点：
>
> 第一是**自主性**。普通 LLM 是被动响应，而 Agent 可以根据目标主动规划多步任务。
>
> 第二是**行动能力**。普通 LLM 只能生成文本，而 Agent 可以通过工具调用和外部系统交互，比如搜索网页、查数据库、执行代码、发邮件、调用业务 API 等。
>
> 第三是**闭环能力**。普通 LLM 通常是一次性生成答案，而 Agent 是“**感知、规划、行动、反馈、再调整**”的循环。它会观察每一步执行结果，如果失败或者信息不足，就重新规划或换一种方式继续执行。
>
> 需要注意的是，**工具调用本身不等于 Agent**。工具只是 Agent 的能力之一。真正的 Agent 关键在于它能不能自主决定什么时候用工具、用哪个工具、怎么根据工具结果调整后续步骤。
>
> 从工程上看，LLM 更像 Agent 的“**大脑**”，负责理解、推理、规划和决策；真正执行工具的是外部代码。模型输出工具调用意图和参数，系统执行工具，再把结果反馈给模型，形成一个完整的执行闭环。

---

## 简短版标准答案

如果面试官希望你快速回答，可以说：

> Agent 是一个具备**自主规划和行动能力**的 AI 系统。普通大模型主要是被动问答，输入问题后输出文本；而 Agent 面向的是**目标完成**，它会把复杂任务拆成多步，自己决定下一步做什么、调用什么工具，并根据工具返回结果继续调整。
>
> 所以 Agent 的核心不是“能调用工具”，而是“**自主闭环**”。它的典型流程是：**感知环境、规划任务、调用工具执行、观察结果、再调整计划**。工具调用、记忆机制、多步推理和自我纠错共同构成了 Agent 的能力。
>
> 简单说，普通 LLM 是“**会回答的大脑**”，Agent 是“**能基于大脑去行动并完成目标的系统**”。
