## 面试标准答案

可以这样回答：

> 我理解一个完整的 Agent 系统，通常由四个核心组件构成：**LLM 核心、工具系统、记忆系统和规划模块**。
>
> **LLM 是 Agent 的大脑**，负责理解用户目标、分析当前状态、做出决策，比如判断下一步是继续推理、调用工具，还是输出最终答案。这里面除了模型本身，System Prompt 也很关键，它相当于给 Agent 定义角色、行为边界和输出规范。
>
> **工具系统是 Agent 和外部世界交互的入口**。LLM 本身不能联网、不能查数据库、不能执行代码，所以需要通过工具来扩展能力，比如搜索引擎、数据库查询、代码执行器、文件读取、邮件 API、业务 API 等。需要注意的是，模型并不真正执行工具，它只是根据工具描述决定调用哪个工具、传什么参数，真正执行的是外部代码，然后把结果再反馈给模型。
>
> **记忆系统负责让 Agent 保持状态**，否则 Agent 做复杂任务时会像“失忆”一样，不知道前面发生过什么。记忆一般分为短期记忆和长期记忆。短期记忆通常放在 context window 或 Agent state 里，用来保存当前任务的中间步骤、工具返回结果和对话上下文；长期记忆通常用向量数据库、关系数据库等方式持久化，用来保存用户偏好、历史任务结果、项目背景等跨会话信息，需要时通过检索取回。
>
> **规划模块负责把复杂目标拆解成可执行步骤**。比如用户让 Agent 写一份竞品分析报告，Agent 不能直接生成结果，而是要先拆成搜索竞品信息、提取关键指标、做对比分析、组织报告结构等步骤。规划模块底层依赖 LLM 的推理能力，常见方式包括 CoT、Plan-and-Execute、ReAct 等。Plan-and-Execute 是先规划再执行，结构清晰；ReAct 是边思考边行动，可以根据工具反馈动态调整，更灵活。
>
> 这四个模块组合起来，才构成了一个真正能完成复杂任务的 Agent：**LLM 负责决策，规划模块负责拆任务，工具系统负责执行动作，记忆系统负责保存状态和历史经验**。它们共同形成“规划 → 决策 → 执行 → 观察结果 → 更新记忆 → 再决策”的闭环。

## 简短版标准答案

> Agent 的基本架构主要包括四个核心组件：**LLM、Tools、Memory 和 Planning**。
>
> **LLM 是大脑**，负责理解任务、推理和决策；**Tools 是手脚**，负责和外部系统交互，比如搜索、查库、执行代码、调 API；**Memory 是记忆系统**，负责保存当前任务状态和历史经验，通常分为短期记忆和长期记忆；**Planning 是规划模块**，负责把复杂目标拆解成可执行步骤，并在执行过程中根据反馈调整计划。
>
> 所以一个 Agent 不是简单的“大模型 + 工具”，还必须有记忆和规划能力。没有记忆，它做长任务会丢上下文；没有规划，它面对复杂目标就不知道如何拆解；没有工具，它无法真正行动；没有 LLM，就没有统一的理解和决策核心。
>
> 从执行流程上看，Agent 通常是：**接收用户目标 → 规划任务步骤 → LLM 判断下一步动作 → 调用工具执行 → 把结果写入记忆 → 根据反馈继续决策 → 最终输出结果**。

## 更口语化的面试回答

> 我一般会把 Agent 类比成一个小团队。
>
> **LLM 像老板或者大脑**，负责判断现在要做什么；**规划模块像项目经理**，负责把一个大目标拆成多个小任务；**工具系统像执行团队**，真正去搜索、查数据库、调用 API；**记忆系统像档案室**，负责记录当前任务进展和历史经验。
>
> 比如用户让 Agent 做竞品分析，规划模块会先拆任务，LLM 判断第一步要搜索哪些竞品信息，然后调用搜索工具，搜索结果回来后写入短期记忆。接着 LLM 再根据已有结果判断下一步是继续搜索、提取指标，还是开始生成报告。如果涉及用户以前的偏好或历史项目，还可以从长期记忆里检索相关信息。
>
> 所以 Agent 的核心不是只有工具调用，而是这四个模块形成了一个闭环：**会规划、会行动、会记忆、会根据反馈调整**。

# 组件拆解版

## 1. LLM 核心

### 作用

- 理解用户输入；
- 分析当前任务状态；
- 做下一步决策；
- 判断是否需要调用工具；
- 判断任务是否完成；
- 生成最终答案。

### 面试重点

> LLM 是 Agent 的“大脑”，但不是整个 Agent。它负责决策，不负责真正执行工具。

---

## 2. 工具系统 Tools

### 作用

- 连接外部世界；
- 弥补 LLM 不能行动、知识不实时的问题；
- 执行搜索、查库、发邮件、代码执行、调用 API 等操作。

### 面试重点

> 工具调用是“决策和执行分离”的。  
> LLM 决定调用什么工具、传什么参数；外部程序真正执行工具，并把结果返回给 LLM。

---

## 3. 记忆系统 Memory

### 作用

- 保存当前任务上下文；
- 保存工具调用结果；
- 保存历史任务经验；
- 支持跨轮、跨任务的信息复用。

### 分类

| 记忆类型 | 存储内容 | 常见实现 |
|---|---|---|
| 短期记忆 | 当前任务上下文、中间步骤、工具返回结果 | context window、message history、Agent state |
| 长期记忆 | 用户偏好、历史任务、项目背景、重要知识 | 向量数据库、关系数据库、文档库、图数据库 |

### 面试重点

> 记忆不等于上下文。上下文只是短期记忆的一种实现，长期记忆通常需要持久化存储和检索机制。

---

## 4. 规划模块 Planning

### 作用

- 把复杂目标拆解成多个步骤；
- 决定任务执行顺序；
- 根据执行结果动态调整计划。

### 常见方式

| 方式 | 特点 |
|---|---|
| CoT | 让模型一步步推理 |
| Plan-and-Execute | 先生成完整计划，再按步骤执行 |
| ReAct | 边思考、边行动、边观察结果，适合动态调整 |
| ToT | 展开多条推理路径，选择较优路径，成本更高 |

### 面试重点

> 规划模块决定 Agent 能不能处理复杂任务。简单任务可以一步完成，复杂任务必须拆解，否则容易遗漏步骤或直接生成空泛答案。

---

# Agent 运行流程

```text
用户目标
  ↓
规划模块拆解任务
  ↓
LLM 判断下一步动作
  ↓
如果需要外部信息或操作，就调用工具
  ↓
工具执行后返回结果
  ↓
结果写入短期记忆
  ↓
LLM 根据记忆和反馈继续判断下一步
  ↓
循环执行，直到任务完成
  ↓
输出最终答案
```

# 面试官可能追问 1：为什么不能只说 LLM + Tools？

可以这样回答：

> 因为 **LLM + Tools** 只能说明 Agent 有了**决策能力**和**行动能力**，但还不够完整。
>
> 如果没有**记忆系统**，Agent 在多步任务中就无法稳定保存中间状态，任务执行到后面可能忘记前面做过什么。
>
> 如果没有**规划模块**，Agent 面对复杂目标时就不知道怎么拆解，只能一步到位生成答案，容易遗漏信息。
>
> 所以完整 Agent 至少要包括 **LLM、Tools、Memory、Planning** 四个模块。

---



# 面试官可能追问 2：短期记忆和长期记忆有什么区别？

可以这样回答：

> 短期记忆主要服务于**当前任务**，通常存在上下文窗口或者 Agent state 里，比如当前任务目标、历史对话、工具返回结果、中间推理步骤等。它的特点是访问快，但容量有限，任务结束后通常会清空。
>
> 长期记忆是**跨任务、跨会话**保存的信息，比如用户偏好、历史任务结果、业务知识、项目背景等。它通常会持久化到向量数据库或关系数据库中，使用时通过语义检索或结构化查询取回。
>
> 简单说，短期记忆解决“**当前任务做到哪了**”，长期记忆解决“**过去有哪些经验可以复用**”。

---

# 面试官可能追问 3：工具调用时模型和代码分别负责什么？

可以这样回答：

> 模型负责**决策**，代码负责**执行**。
>
> 工具通常会提前定义好名称、描述和参数 schema。LLM 根据这些描述判断是否需要调用工具，以及调用哪个工具、传入什么参数。然后外部程序解析模型输出的 tool call，真正去执行函数或 API。执行结果再作为 observation 返回给 LLM，让它继续推理。
>
> 所以工具调用不是模型自己真的在执行，而是模型输出一个**结构化的调用意图**，外部系统完成动作。

---

# 面试官可能追问 4：Planning 有哪些常见模式？

可以这样回答：

> 常见有两类。
>
> 第一类是 **Plan-and-Execute**，先让模型生成完整计划，再按步骤执行。优点是结构清晰、可控性强，适合流程比较明确、需要人工审核的任务；缺点是如果中间结果和预期不一致，原计划可能需要重新调整。
>
> 第二类是 **ReAct**，也就是 **Reasoning and Acting**，边思考边行动。每一步模型都会根据当前 observation 决定下一步 action。优点是灵活，适合开放性强、信息不确定的任务；缺点是容易局部最优，可能走偏，需要通过约束、反思或阶段性总结来控制。
>
> 实际工程里通常会结合使用：先做一个粗略计划确定方向，执行过程中再用 ReAct 根据反馈动态调整。

# 最终背诵版

> Agent 的基本架构一般包括四个核心组件：**LLM、工具系统、记忆系统和规划模块**。
>
> LLM 是 Agent 的大脑，负责理解任务、推理和决策；工具系统是 Agent 的手脚，负责和外部世界交互，比如搜索、查数据库、执行代码、调用 API；记忆系统负责保存状态，短期记忆保存当前任务的上下文和中间结果，长期记忆保存跨会话的用户偏好、历史经验和业务知识；规划模块负责把复杂目标拆解成可执行步骤，并根据执行反馈调整计划。
>
> 这四个模块组合起来，形成了 Agent 的执行闭环：**规划任务、做出决策、调用工具、观察结果、更新记忆、继续决策**。所以 Agent 不是简单的“大模型 + 工具”，真正完整的 Agent 必须具备规划、行动、记忆和反馈调整能力。
