
## 一、面试标准回答

面试官问：**大模型生成文本时有哪些解码策略？贪心、Beam Search、采样分别什么时候用？**

可以这样回答：

大模型是自回归生成的，每生成一个 token，模型都会输出一个词表大小的概率分布。  
**解码策略**解决的就是：

> **怎么从这个概率分布里选出下一个 token。**

主流解码策略可以分成两大类。

第一类是**确定性策略**，相同输入通常得到相同输出。

### 1. 贪心解码

贪心解码每一步都选概率最高的 token。

优点是：

- 简单；
- 稳定；
- 可复现；
- 适合代码生成、SQL 生成、JSON 抽取这类精确任务。

缺点是：

- 容易保守；
- 缺乏多样性；
- 容易陷入重复循环。

### 2. Beam Search

Beam Search 是贪心的扩展，每一步保留 `B` 条候选路径，最后选择整体概率最高的序列。

它在机器翻译时代很常用，因为翻译任务通常存在相对明确的最优译文。  
但在 LLM 开放式生成任务里，Beam Search 反而不常用。

原因是开放式对话、写作、头脑风暴没有唯一最优答案。  
Beam Search 追求整体概率最高，往往会生成最安全、最保守、最 boring 的回答，还容易重复。  
同时 Beam Search 要维护多条候选序列，会带来更高的计算成本和 KV Cache 显存开销。

第二类是**随机性策略**，也就是采样。

### 3. Temperature 采样

Temperature 控制概率分布的锐度。

~~~text
Temperature 越低：分布越尖锐，输出越稳定
Temperature 越高：分布越平坦，输出越发散
~~~

当 `Temperature = 0` 时，基本等价于每次选择概率最高的 token，也就是接近贪心。

### 4. Top-K 采样

Top-K 每一步只从概率最高的 `K` 个 token 中采样，直接截断低概率长尾 token。

### 5. Top-P 采样

Top-P 又叫 Nucleus Sampling。  
它不是固定保留 K 个 token，而是从高到低累加概率，直到累计概率超过 `P`，然后只在这个候选集合里采样。

Top-P 的好处是候选集大小会随概率分布自动变化：

~~~text
分布尖锐：候选集小
分布平坦：候选集大
~~~

实际工程中可以这样选：

~~~text
有标准答案的任务：贪心 / Temperature=0
需要稳定但自然的对话：Temperature=0.7 + Top-P=0.9
创意写作 / 头脑风暴：Temperature=1.0~1.2 + Top-P=0.95
数学推理：单次低温，或者多次采样 + Self-Consistency 投票
~~~

一句话总结：

> **解码策略本质是从概率分布里选 token；精确任务偏确定性，开放任务偏采样，多样性任务提高 temperature 和 top-p。**

## 二、这篇文章的核心观点

这篇文章主要讲的是：

> **解码策略不是简单背参数，而是要理解不同任务到底需要“唯一最优答案”还是“多种合理答案”。**

核心主线：

~~~text
模型每步输出一个概率分布
    ↓
解码策略决定怎么选下一个 token
    ↓
确定性策略追求最可能结果
    ↓
采样策略追求合理多样性
    ↓
不同任务按确定性 / 多样性需求选择策略
~~~

最关键的判断是：

~~~text
任务有标准答案：用贪心或低温
任务有多种合理答案：用采样
任务鼓励发散：提高温度和 Top-P
~~~

## 三、解码策略的本质

LLM 是自回归生成模型。

每一步都会根据当前上下文输出一个概率分布：

~~~text
P(next_token | previous_tokens)
~~~

例如：

~~~text
输入：我今天想吃
~~~

模型可能输出：

~~~text
苹果：0.30
香蕉：0.20
面条：0.15
火锅：0.10
其他 token：...
~~~

解码策略就是：

> **从这个概率分布中选一个 token。**

不同策略的差别在于：

~~~text
是直接选概率最高
还是保留多个候选
还是按概率随机采样
是否截断低概率 token
是否调整分布锐度
~~~

---

## 四、确定性策略：贪心解码

贪心解码的规则非常简单：

> **每一步都选择概率最高的 token。**

例如：

~~~text
苹果：0.30
香蕉：0.20
面条：0.15
~~~

贪心会直接选择：

~~~text
苹果
~~~

然后继续下一步。

---

## 五、贪心解码的优缺点

### 1. 优点

贪心解码的优点：

- 实现简单；
- 速度快；
- 输出稳定；
- 可复现；
- 适合精确任务。

适合：

~~~text
代码生成
SQL 生成
JSON 抽取
信息抽取
格式化输出
~~~

这些任务通常有比较明确的标准答案。

---

### 2. 缺点

贪心的缺点是：

> **太保守。**

它每一步都走最稳的路，容易导致：

- 回答单调；
- 缺乏创造性；
- 容易重复；
- 陷入复读循环。

例如：

~~~text
He is so cute. He is so cute. He is so cute...
~~~

原因是模型一旦生成某种高概率模式，后续上下文会继续强化这个模式。

所以贪心不适合：

~~~text
创意写作
开放对话
头脑风暴
故事生成
~~~

---

## 六、确定性策略：Beam Search

Beam Search 可以理解为：

> **贪心解码的多候选版本。**

贪心每一步只保留 1 条路径。

Beam Search 每一步保留 `B` 条概率最高的路径，其中 `B` 叫 beam width。

例如 `B = 3`：

~~~text
第 1 步：保留 3 个候选
第 2 步：每个候选继续展开，再保留总分最高的 3 条
第 3 步：继续展开
最后：选择整体概率最高的完整序列
~~~

---

## 七、Beam Search 为什么翻译时代很强？

Beam Search 在机器翻译时代非常常用。

原因是翻译任务通常有：

> **相对明确的最优译文。**

例如：

~~~text
I love you.
~~~

合理译文通常集中在：

~~~text
我爱你
我喜欢你
~~~

这类任务中，寻找整体概率最高的序列是合理的。

所以 Beam Search 的目标：

~~~text
找到整体概率最高的序列
~~~

和翻译任务比较匹配。

---

## 八、为什么 LLM 时代 Beam Search 失宠？

LLM 的开放式生成任务不同。

比如：

~~~text
写一段自我介绍
讲一个故事
分析一个项目
回答一个开放问题
~~~

这些任务没有唯一最优答案，而是有很多合理答案。

Beam Search 的问题是：

> **它过度追求高概率序列。**

而高概率序列往往是：

- 最保守；
- 最常见；
- 最无聊；
- 最容易重复。

所以在开放式生成中，Beam Search 容易产生：

~~~text
boring answer
repetition loop
~~~

另外，Beam Search 还会带来工程成本：

- 计算量约增加 `B` 倍；
- KV Cache 要维护多条候选路径；
- 显存占用更高；
- 和一些推理优化不够匹配。

所以现代 LLM 对话 / 写作通常不优先用 Beam Search。

---

## 九、采样策略：用随机性换多样性

采样策略的核心思想是：

> **不总是选最高概率 token，而是按概率分布随机抽样。**

这样模型每次输出可能不同。

采样适合：

- 日常对话；
- 创意写作；
- 头脑风暴；
- 开放式问答；
- 多样化生成。

但普通采样有一个问题：

> **可能抽到长尾低概率 token。**

低概率 token 有时是不合理的，可能导致输出突然跑偏。

所以需要：

~~~text
Temperature
Top-K
Top-P
~~~

来控制采样范围和随机性。

---

## 十、Temperature：控制随机性强弱

Temperature 用来调整概率分布的锐度。

直觉上：

~~~text
Temperature 低：更确定，更保守
Temperature 高：更随机，更发散
~~~

常见理解：

~~~text
T = 0：接近贪心
T = 0.2：非常稳定
T = 0.7：自然对话常用
T = 1.0：标准采样
T > 1.0：更发散，更有创意
~~~

如果 temperature 太高，模型可能：

- 胡言乱语；
- 偏离主题；
- 生成低质量内容。

如果 temperature 太低，模型可能：

- 太保守；
- 重复；
- 缺乏创造性。

---

## 十一、Top-K：固定截断候选集

Top-K 的规则是：

> **每一步只保留概率最高的 K 个 token，然后在这些 token 中采样。**

例如：

~~~text
K = 50
~~~

表示：

~~~text
只从概率最高的 50 个 token 中采样
其他 token 直接丢弃
~~~

优点：

- 简单；
- 能截断长尾噪声；
- 防止抽到极低概率 token。

缺点：

> **K 是固定的，不够自适应。**

有时概率分布很尖锐，只需要几个候选；  
有时概率分布很平坦，需要更多候选。  
固定 K 不能根据分布自动调整。

---

## 十二、Top-P：自适应截断候选集

Top-P 又叫：

~~~text
Nucleus Sampling
~~~

它的规则是：

> **从高概率 token 开始累加，直到累计概率超过 P，然后只在这个集合中采样。**

例如：

~~~text
Top-P = 0.9
~~~

表示保留一个最小候选集，使得这些 token 的累计概率达到 90%。

如果分布很尖锐：

~~~text
前几个 token 就达到 0.9
候选集很小
~~~

如果分布很平坦：

~~~text
需要很多 token 才达到 0.9
候选集变大
~~~

所以 Top-P 比 Top-K 更自适应，也更常用于 LLM API。

---

## 十三、Top-K 和 Top-P 的区别

| 对比项 | Top-K | Top-P |
|---|---|---|
| 截断方式 | 固定保留 K 个 token | 保留累计概率达到 P 的 token |
| 候选集大小 | 固定 | 动态变化 |
| 自适应能力 | 弱 | 强 |
| 常见程度 | 较常见 | LLM API 更常用 |
| 适合场景 | 简单控制长尾 | 开放式生成常用 |

可以记：

~~~text
Top-K：固定人数入围
Top-P：按累计概率入围
~~~

---

## 十四、实际选型：什么任务用什么策略？

| 任务类型 | 推荐策略 | 典型参数 | 原因 |
|---|---|---|---|
| 代码生成 | 贪心 / 低温采样 | Temperature=0~0.2 | 代码结构明确，要稳定 |
| SQL 生成 | 贪心 | Temperature=0 | 输出错一个字符就可能报错 |
| JSON 抽取 | 贪心 | Temperature=0 | 格式严格，需要可复现 |
| 信息抽取 | 贪心 / 低温 | Temperature=0~0.2 | 追求准确稳定 |
| 数学推理 | 低温 / Self-Consistency | T=0 或 T=0.7 多次采样 | 单次稳定，多次投票更准 |
| 日常对话 | Top-P 采样 | T=0.7, Top-P=0.9 | 自然且不太发散 |
| 创意写作 | 高温 Top-P | T=1.0~1.2, Top-P=0.95 | 鼓励多样性和惊喜 |
| 头脑风暴 | 高温 Top-P | T=1.2, Top-P=0.95 | 越发散越好 |
| 机器翻译 | 贪心 / 低温 / Beam Search | T=0~0.3 | 有相对标准答案 |

---

## 十五、选型口诀

可以这样记：

~~~text
任务有标准答案：
贪心 / Temperature=0

任务需要稳定但自然：
Temperature=0.7 + Top-P=0.9

任务鼓励多样性：
Temperature=1.0~1.2 + Top-P=0.95

任务是数学推理：
低温单次生成
或较高温多次采样 + 多数投票

任务是翻译：
低温 / Beam Search 仍可能有用
~~~

## 十六、进阶策略：Self-Consistency

Self-Consistency 常用于数学、逻辑推理任务。

核心思想是：

> **对同一个问题采样多条推理路径，最后对最终答案投票。**

流程：

~~~text
同一个问题
    ↓
用较高 Temperature 采样 N 次
    ↓
得到 N 条推理路径
    ↓
提取最终答案
    ↓
多数投票
~~~

为什么有效？

因为正确答案往往可以通过多种推理路径得到；错误答案更随机，不容易集中到同一个结果。

缺点是：

> **成本变成 N 倍。**

---

## 十七、进阶策略：推测解码

推测解码是推理加速技术，不是改变输出质量的主要策略。

核心思想：

> **用小模型先草拟多个 token，再用大模型一次性验证。**

流程：

~~~text
小模型快速生成几个 token
    ↓
大模型并行验证这些 token
    ↓
如果一致就接受
    ↓
不一致以大模型为准
~~~

好处：

- 减少大模型逐 token 解码次数；
- 提升推理速度；
- 输出结果可以保持和大模型一致。

它属于：

> **解码加速策略。**

---

~~~

## 十九、面试官可能继续追问

### 1. 贪心和 Temperature=0 是一回事吗？

工程上通常接近。

Temperature 趋近 0 时，概率最高 token 的概率会接近 1，因此基本等价于贪心。

但严格实现上，不同框架对 `temperature=0` 的处理可能不同，有些会直接切换到 greedy。

---

### 2. Beam Search 是确定性还是随机性？

Beam Search 是确定性策略。

它不是随机采样，而是保留多条高概率路径，最后选整体概率最高的序列。

---

### 3. 为什么 LLM 很少用 Beam Search？

因为开放式生成没有唯一最优答案。

Beam Search 追求最高概率序列，容易输出保守、重复、乏味的内容。  
同时它还会带来更高计算和 KV Cache 成本。

---

### 4. Top-K 和 Top-P 有什么区别？

Top-K 固定保留概率最高的 K 个 token。

Top-P 保留累计概率达到 P 的最小 token 集合。

所以：

~~~text
Top-K 是固定截断
Top-P 是自适应截断
~~~

---

### 5. Temperature 控制什么？

Temperature 控制概率分布的锐度。

~~~text
低温：更确定，更稳定
高温：更随机，更发散
~~~

---

### 6. 什么任务适合 Temperature=0？

适合：

- 代码生成；
- SQL 生成；
- JSON 抽取；
- 信息抽取；
- 单元测试；
- 要求可复现的批处理任务。

---

### 7. 创意写作用什么参数？

一般可以用：

~~~text
Temperature=1.0~1.2
Top-P=0.95
~~~

这样输出更有多样性和惊喜。

---

### 8. 数学推理怎么选？

单次回答可以低温或贪心，保证稳定。

如果追求更高准确率，可以用：

~~~text
Self-Consistency
~~~

也就是多次采样推理路径，再对最终答案投票。

---

## 二十、记忆版总结

这篇文章可以压缩成一句话：

> **解码策略决定模型每一步怎么从概率分布里选 token；有标准答案用确定性策略，开放生成用采样，推理任务可以用多次采样投票。**

进一步记：

~~~text
贪心：
每步选最高概率
稳定、可复现
适合代码、SQL、JSON、信息抽取
缺点是保守、重复

Beam Search：
保留多条候选
适合翻译等有相对标准答案的任务
LLM 开放生成中容易 boring、重复、显存贵

Temperature：
控制随机性
低温稳定
高温发散

Top-K：
固定保留 K 个候选

Top-P：
保留累计概率达到 P 的候选集
更自适应

实际选型：
标准答案：T=0
自然对话：T=0.7, Top-P=0.9
创意写作：T=1.0~1.2, Top-P=0.95
数学推理：低温或 Self-Consistency
~~~

最重要的一句话：

> **解码策略不是参数玄学，而是任务对“确定性”和“多样性”的取舍。**
