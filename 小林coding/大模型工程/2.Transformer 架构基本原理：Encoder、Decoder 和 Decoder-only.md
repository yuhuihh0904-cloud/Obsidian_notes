
## 一、面试标准回答

面试官问：**讲讲 Transformer 架构基本原理？Encoder 和 Decoder 是什么？为什么现在主流大模型都用 Decoder-only？**

可以这样回答：

Transformer 是一种基于 **Self-Attention** 的序列建模架构。它解决了 RNN 的两个核心问题：

第一，**RNN 必须顺序计算，无法充分并行**。  
RNN 处理第 `t` 个 token 时，必须等第 `t-1` 个 token 计算完，所以训练长序列时效率很低。

第二，**RNN 难以建模长距离依赖**。  
虽然 LSTM、GRU 用门控机制缓解了梯度消失，但当序列很长时，早期 token 的信息传到后面仍然容易衰减。

Transformer 用 Self-Attention 解决这两个问题。  
Self-Attention 让每个 token 都能直接和序列中任意位置的 token 建立联系，并且所有位置的注意力可以并行计算。每个 token 会从输入 embedding 经过三个独立矩阵投影得到 `Q`、`K`、`V`：

~~~text
Q：Query，表示我想找什么
K：Key，表示我有什么标签
V：Value，表示我的实际内容
~~~

然后通过：

~~~text
Attention(Q, K, V) = softmax(QK^T / √d_k) V
~~~

计算注意力结果。这里除以 `√d_k` 是为了防止点积值过大，让 softmax 变成接近 one-hot，从而导致梯度变小、训练不稳定。

Encoder 和 Decoder 的区别主要在于注意力是否能看未来信息：

- **Encoder-only**：双向注意力，每个 token 可以看前后文，适合理解类任务，比如 BERT；
- **Decoder-only**：因果注意力，每个 token 只能看前面的 token，天然适合生成任务，比如 GPT、Claude、Qwen；
- **Encoder-Decoder**：Encoder 双向理解输入，Decoder 单向生成输出，适合翻译、摘要等输入输出不同的任务，比如 T5、BART。

现在主流大模型选择 Decoder-only，核心原因是：

> **预测下一个 token 这个训练目标极其统一。**

问答、写作、翻译、代码生成、推理都可以统一成：

~~~text
给定前文，预测下一个 token
~~~

这个目标可以直接利用海量无标注文本做自监督训练，容易 scale up。随着模型规模、数据规模和算力提升，Decoder-only 架构在这个统一目标下涌现出越来越强的生成、推理和代码能力。

一句话总结：

> **Transformer 用 Self-Attention 解决了 RNN 慢和长距离依赖弱的问题；Decoder-only 赢在训练目标统一、数据利用效率高、最适合大规模生成式预训练。**

## 二、这篇文章的核心观点

这篇文章主要讲的是：

> **理解 Transformer 不能只背“Attention is All You Need”，而要讲清楚 RNN 卡在哪里、Self-Attention 怎么解决、三种 Transformer 架构为什么最后 Decoder-only 成为主流。**

主线可以压缩成：

~~~text
RNN 有两个致命问题
    ↓
Transformer 用 Self-Attention 解决
    ↓
Transformer 分成 Encoder-only、Decoder-only、Encoder-Decoder
    ↓
大模型时代 Decoder-only 因为 next token prediction 目标统一而胜出
~~~

---

## 三、Transformer 之前：RNN 的两个致命缺陷

### 1. 顺序计算，无法并行

RNN 处理序列时是从左到右一步步计算：

~~~text
h1 -> h2 -> h3 -> ... -> hn
~~~

第 `t` 步必须等第 `t-1` 步完成。

这导致：

- 训练速度慢；
- 很难充分利用 GPU 并行能力；
- 长序列训练效率低。

---

### 2. 长距离信息衰减

RNN 理论上能把前面信息传到后面，但实际中长距离依赖很难学。

比如第 1 个 token 和第 800 个 token 有关系，信息要经过很多步传递：

~~~text
token_1 -> token_2 -> ... -> token_800
~~~

中间不断变换，梯度容易衰减。

LSTM、GRU 通过门控机制缓解这个问题，但没有从根本上解决。

---

## 四、Self-Attention 的核心直觉

Self-Attention 的核心是：

> **让每个 token 直接和序列中所有 token 建立联系。**

也就是说，不再像 RNN 那样一层层传递信息，而是直接计算：

~~~text
当前 token 和其他所有 token 的相关性
~~~

然后根据相关性加权汇总其他 token 的信息。

这样有两个好处：

- 所有 token 可以并行计算；
- 任意两个位置之间都有直接连接，长距离依赖更容易建模。

---

## 五、Q、K、V 是什么？

Self-Attention 里每个 token 会产生三个向量：

~~~text
Q = Query：我想找什么
K = Key：我有什么特征 / 标签
V = Value：我的实际内容
~~~

可以用图书馆检索类比：

~~~text
Q：你输入的搜索关键词
K：每本书的标签
V：每本书的内容
~~~

检索过程就是：

~~~text
用 Q 去匹配所有 K
    ↓
得到相关性分数
    ↓
按分数加权汇总 V
~~~

---

## 六、Q/K/V 是怎么来的？

Q、K、V 不是凭空来的，而是从同一个输入 `X` 通过三个不同的线性矩阵投影得到的：

~~~text
Q = X @ W_Q
K = X @ W_K
V = X @ W_V
~~~

其中：

- `X` 是输入 token embedding；
- `W_Q`、`W_K`、`W_V` 是可训练参数；
- 三个矩阵相互独立。

为什么要三个不同矩阵？

因为模型需要从不同角度理解同一个 token：

~~~text
Q：这个 token 想关注什么
K：这个 token 能被什么关注
V：这个 token 真正提供什么内容
~~~

如果 Q/K/V 都直接等于 X，模型表达能力会弱很多。

---

## 七、Attention 公式

Self-Attention 的公式是：

~~~text
Attention(Q, K, V) = softmax(QK^T / √d_k) V
~~~

可以拆成三步：

### 1. 计算相关性

~~~text
QK^T
~~~

每个 token 的 Query 和所有 token 的 Key 做点积，得到注意力分数。

---

### 2. 缩放并归一化

~~~text
softmax(QK^T / √d_k)
~~~

除以 `√d_k` 后再 softmax，得到注意力权重。

---

### 3. 加权求和

~~~text
attention_weights @ V
~~~

根据注意力权重，对所有 Value 加权求和，得到当前 token 的新表示。

---

## 八、为什么要除以 √d_k？

这是面试常问细节。

如果 `d_k` 很大，Q 和 K 点积的数值会变得很大。

点积值过大后，softmax 会变得非常尖锐：

~~~text
最大值接近 1
其他值接近 0
~~~

也就是接近 one-hot。

这样会导致：

- 注意力分布过于极端；
- 梯度变小；
- 训练不稳定。

除以 `√d_k` 的作用是：

> **把点积数值缩放到合理范围，让 softmax 不至于饱和，保证梯度稳定。**

---

## 九、Multi-Head Attention：多头注意力

单个注意力头只能从一个角度看 token 之间的关系。

但语言关系有很多种：

~~~text
主谓关系
动宾关系
指代关系
位置关系
语义相关
格式结构
~~~

所以 Multi-Head Attention 会并行使用多个注意力头：

~~~text
Head 1：关注语法关系
Head 2：关注指代关系
Head 3：关注实体关系
Head 4：关注局部上下文
...
~~~

每个头学习不同子空间里的关系，最后把多个头的结果拼接起来。

好处是：

> **让模型从多个角度理解 token 之间的关系。**

---

## 十、为什么需要位置编码？

Self-Attention 本身不关心顺序。

如果没有位置编码，模型只知道有哪些 token，不知道谁在前、谁在后。

例如：

~~~text
我打你
你打我
~~~

这两个句子的 token 相同，但顺序不同，意思完全不同。

所以 Transformer 需要加入位置编码，让模型知道 token 的位置信息。

常见位置编码包括：

~~~text
sin/cos 绝对位置编码
RoPE 旋转位置编码
ALiBi
~~~

---

## 十一、FFN 的作用

Transformer Block 里除了 Attention，还有 FFN：

~~~text
Attention
    ↓
FFN
~~~

Attention 主要负责：

> **token 之间的信息交互。**

FFN 主要负责：

> **对每个 token 的表示做非线性变换。**

可以理解为：

- Attention 负责“从上下文拿信息”；
- FFN 负责“消化和加工信息”。

很多研究也认为 FFN 层存储了大量事实知识，可以粗略理解为模型的“记忆仓库”。

---

## 十二、三种 Transformer 架构

### 1. Encoder-only

代表模型：

~~~text
BERT
~~~

特点：

- 双向注意力；
- 每个 token 可以同时看左边和右边；
- 适合理解类任务。

典型任务：

~~~text
文本分类
命名实体识别
语义相似度
检索 embedding
```

预训练目标通常是：

~~~text
MLM：Masked Language Modeling
~~~

也就是随机遮住一些 token，让模型预测被遮住的词。

---

### 2. Decoder-only

代表模型：

~~~text
GPT
Claude
Qwen
LLaMA
DeepSeek
~~~

特点：

- 使用因果 mask；
- 每个 token 只能看自己前面的 token；
- 不能看未来 token；
- 天然适合生成。

预训练目标是：

~~~text
CLM：Causal Language Modeling
预测下一个 token
~~~

例如：

~~~text
输入：我喜欢吃
目标：苹果
~~~

---

### 3. Encoder-Decoder

代表模型：

~~~text
T5
BART
原始 Transformer 翻译模型
~~~

特点：

- Encoder 双向理解输入；
- Decoder 单向生成输出；
- Decoder 通过 Cross-Attention 读取 Encoder 输出。

适合：

~~~text
翻译
摘要
问答
文本改写
~~~

这类输入和输出结构不同的任务。

---

## 十三、为什么 Decoder-only 赢了？

这是这篇文章最重要的部分。

### 1. 训练目标极其统一

Decoder-only 的训练目标是：

~~~text
预测下一个 token
~~~

这个目标非常简单，但可以统一很多任务：

~~~text
问答：根据问题继续生成答案
写作：根据开头继续写文章
翻译：根据翻译指令继续生成译文
代码：根据需求继续生成代码
推理：根据题目继续生成推理过程和答案
~~~

所以它不需要为不同任务设计不同训练目标。

---

### 2. 可以利用海量无标注文本

预测下一个 token 是天然的自监督任务。

互联网文本本身就可以构造训练样本：

~~~text
前面的 token：输入
下一个 token：标签
~~~

不需要人工标注。

这使得 Decoder-only 非常适合大规模训练。

---

### 3. 更容易 scale up

Decoder-only 架构简单，目标统一，很适合扩大：

~~~text
模型参数
训练数据
训练算力
~~~

规模上去后，模型会逐渐表现出：

- 指令理解；
- 上下文学习；
- 代码能力；
- 多步推理；
- 跨语言迁移。

所以 Decoder-only 在大模型时代更容易形成通用能力。

---

### 4. 统一了任务接口

Decoder-only 的接口就是：

~~~text
Prompt + 续写
~~~

所有任务都可以变成：

~~~text
给模型一段上下文，让它继续生成
~~~

这比 Encoder-only 或 Encoder-Decoder 更适合通用对话和生成式 AI 产品。

---

## 十四、三种架构对比表

| 架构 | 代表模型 | 注意力方式 | 训练目标 | 适合任务 |
|---|---|---|---|---|
| Encoder-only | BERT | 双向注意力 | MLM | 理解、分类、NER、Embedding |
| Decoder-only | GPT、Claude、Qwen | 因果注意力 | 预测下一个 token | 生成、问答、代码、推理 |
| Encoder-Decoder | T5、BART | Encoder 双向 + Decoder 单向 | Seq2Seq | 翻译、摘要、改写 |

## 十六、面试官可能继续追问

### 1. Transformer 为什么比 RNN 更适合长序列？

因为 Self-Attention 让任意两个 token 之间可以直接计算相关性，不需要像 RNN 那样一步步传递。

所以长距离依赖不会随着序列长度增加而层层衰减。

---

### 2. Attention 中 Q/K/V 分别是什么？

可以这样回答：

~~~text
Q：当前 token 想找什么信息
K：每个 token 提供什么匹配标签
V：每个 token 真正携带的内容
~~~

模型用 Q 和 K 算相似度，再根据相似度加权汇总 V。

---

### 3. Q/K/V 为什么要用三个不同矩阵？

因为同一个 token 在注意力里有三种角色：

~~~text
作为查询者
作为被匹配对象
作为信息提供者
~~~

三个独立矩阵能让模型从不同角度学习表示，增强表达能力。

---

### 4. 为什么 Attention 要除以 √d_k？

因为 Q 和 K 的维度越大，点积值方差越大。

如果不缩放，softmax 会过于尖锐，接近 one-hot，导致梯度变小、训练不稳定。

除以 `√d_k` 可以稳定 softmax 输入范围。

---

### 5. Encoder-only 为什么适合理解任务？

因为它是双向注意力。

每个 token 可以同时看左边和右边上下文，所以更适合分类、NER、语义匹配等需要完整理解句子的任务。

---

### 6. Decoder-only 为什么适合生成任务？

因为它使用因果 mask。

每个 token 只能看前面的 token，这和文本生成时“根据已生成内容预测下一个 token”的过程完全一致。

---

### 7. 为什么 Decoder-only 成为主流大模型架构？

核心原因有三点：

~~~text
训练目标统一：预测下一个 token
数据来源广：可以直接用海量无标注文本
容易规模化：模型越大，能力越强
~~~

所以它最适合做通用生成式大模型。

---

## 十七、记忆版总结

这篇文章可以压缩成一句话：

> **Transformer 用 Self-Attention 解决了 RNN 顺序计算慢和长距离依赖弱的问题，而 Decoder-only 凭借“预测下一个 token”这个统一目标成为大模型时代的主流架构。**

进一步记：

~~~text
RNN 两个问题：
顺序计算慢
长距离依赖弱

Self-Attention：
每个 token 直接看所有 token
并行计算
长距离直接连接

Q/K/V：
Q 表示想找什么
K 表示能被怎么匹配
V 表示实际内容

除以 √d_k：
防止 softmax 过尖
稳定梯度

三种架构：
Encoder-only：双向理解
Decoder-only：单向生成
Encoder-Decoder：理解输入后生成输出

Decoder-only 赢的原因：
next token prediction 目标统一
海量无标注数据可用
容易 scale up
Prompt + 续写统一所有任务
~~~

最重要的一句话：

> **面试讲 Transformer，不要只背架构图，要讲清楚：RNN 为什么不行，Self-Attention 怎么解决，Decoder-only 为什么在大模型时代胜出。**
