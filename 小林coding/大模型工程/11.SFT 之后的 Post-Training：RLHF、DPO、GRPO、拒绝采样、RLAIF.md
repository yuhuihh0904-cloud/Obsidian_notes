
## 面试标准回答

Post-Training 是一个**上位概念**，指的是模型完成预训练和 SFT 之后，为了进一步提升模型质量、偏好对齐、安全性和推理能力而进行的一系列训练方法。

SFT 的作用是让模型从“文本续写机器”变成“能按指令回答的对话模型”。但是 SFT 之后，模型通常还有两个问题：

1. **只学会了合格回答，不一定学会优质回答**
   - 同一个问题可以有多个合格答案；
   - 但用户更喜欢哪个答案，SFT 本身不一定能学清楚。

2. **安全性和价值观对齐不够**
   - 有些问题不应该直接回答；
   - 有些问题应该拒答；
   - 有些问题需要承认不确定性；
   - 这些能力往往需要额外对齐训练。

所以 SFT 之后还需要 Post-Training。

主流 Post-Training 方法主要包括：

1. **RLHF**
2. **DPO**
3. **GRPO**
4. **拒绝采样**
5. **RLAIF**

它们不是简单的互相替代关系，而是经常组合使用。

---

## 1. RLHF：经典方案，4 模型架构

RLHF 全称是 **Reinforcement Learning from Human Feedback**，也就是基于人类反馈的强化学习。

它的核心流程是：

~~~text
SFT 模型生成多个回答
        ↓
人类标注员对回答做偏好排序
        ↓
训练 Reward Model
        ↓
用 PPO 优化主模型
        ↓
用 Reference Model 做 KL 约束，防止模型跑偏
~~~

RLHF 里通常涉及 4 个模型：

| 模型 | 作用 |
|---|---|
| Policy Model | 当前要训练的主模型 |
| Reference Model | 冻结的 SFT 模型，用来约束主模型别偏离太远 |
| Reward Model | 根据人类偏好给回答打分 |
| Value Model | PPO 中估计状态价值，用来计算优势函数 |

RLHF 的优点是：

- **效果上限高**
- **可以探索 SFT 数据之外的更优回答**
- **早期 ChatGPT、InstructGPT 采用过这类路线**

缺点也很明显：

- 训练流程复杂；
- 要维护多个模型；
- 显存占用高；
- PPO 训练不稳定；
- 超参数敏感；
- 有 reward hacking 风险。

一句话：

**RLHF 是经典但昂贵、复杂、难调的对齐方案。**

---

## 2. DPO：绕过 Reward Model 的简化版 RLHF

DPO 全称是 **Direct Preference Optimization**，也就是直接偏好优化。

它的核心思想是：

**不再显式训练 Reward Model，也不再跑 PPO，而是直接用偏好数据训练模型。**

DPO 数据格式是：

~~~text
{
  "prompt": "如何学习 Python？",
  "chosen": "人类更喜欢的回答",
  "rejected": "人类不喜欢的回答"
}
~~~

DPO 训练时做的事情可以直观理解为：

~~~text
提高模型生成 chosen 的概率
降低模型生成 rejected 的概率
同时用 Reference Model 约束模型不要偏离太远
~~~

DPO 只需要两个模型：

| 模型 | 作用 |
|---|---|
| Policy Model | 当前训练的主模型 |
| Reference Model | 冻结的参考模型 |

所以相比 RLHF，DPO 简化了很多：

| 对比项 | RLHF | DPO |
|---|---|---|
| 是否需要 Reward Model | 需要 | 不需要 |
| 是否需要 PPO | 需要 | 不需要 |
| 是否需要 Value Model | 需要 | 不需要 |
| 训练形式 | 强化学习 | 监督学习 |
| 稳定性 | 较差 | 更好 |
| 工程复杂度 | 高 | 低 |

DPO 的优点：

- 实现简单；
- 训练稳定；
- 显存压力更小；
- 非常适合开源社区和资源有限团队。

DPO 的缺点：

- 效果上限依赖偏好数据质量；
- 探索能力不如精心调优的 RLHF；
- 本质上更像“往已有偏好数据分布靠拢”。

一句话：

**DPO 是把 RLHF 简化成监督学习的偏好优化方法，是开源 Instruct 模型中非常常见的低成本对齐方案。**

---

## 3. GRPO：砍掉 Value Model 的 PPO 进化版

GRPO 全称是 **Group Relative Policy Optimization**，可以理解为 PPO 的改进版。

它的核心创新是：

**砍掉 PPO 里的 Value Model，用同一个问题下多个回答的组内相对分数来估计优势函数。**

PPO 中需要 Value Model 的原因是：

~~~text
优势函数 = 实际奖励 - 预期奖励
~~~

其中“预期奖励”通常由 Value Model 估计。

但是 Value Model 本身也是一个大模型：

- 需要额外训练；
- 需要额外显存；
- 需要额外调参；
- 会增加训练复杂度。

GRPO 的做法是：

~~~text
对同一个 Prompt 采样 G 个回答
        ↓
给每个回答打分
        ↓
计算组内平均分和标准差
        ↓
用组内相对优势替代 Value Model
~~~

公式直觉：

~~~text
A_i = (r_i - mean(r_1, r_2, ..., r_G)) / std(r_1, r_2, ..., r_G)
~~~

其中：

- **r_i**：第 i 个回答的奖励分数；
- **mean**：同组回答的平均奖励；
- **std**：同组回答奖励的标准差；
- **A_i**：第 i 个回答相对于组内平均水平的优势。

也就是说：

~~~text
如果一个回答比同组其他回答好，它的优势就是正的；
如果一个回答比同组其他回答差，它的优势就是负的。
~~~

GRPO 的优点：

- 省掉 Value Model；
- 显存占用更低；
- 训练比 PPO 更简单；
- 仍然保留 RL 的探索能力；
- 特别适合数学、代码这类可验证任务。

为什么特别适合数学和代码？

因为这类任务可以直接判断对错：

~~~text
数学题：答案对就是 1，错就是 0
代码题：单元测试通过就是 1，不通过就是 0
~~~

这种场景甚至可以不训练 Reward Model，直接用规则或验证器给奖励。

一句话：

**GRPO 是一种省掉 Value Model 的强化学习对齐方法，特别适合数学、代码、推理这类可验证任务，也是 DeepSeek R1 这类推理模型路线里非常关键的方法。**

---

## 4. 拒绝采样：生成、筛选、再 SFT

拒绝采样，全称可以叫 **Rejection Sampling Fine-tuning**。

它不是强化学习，而是一种非常简单的迭代式 SFT 方法。

流程是：

~~~text
准备一批 Prompt
        ↓
让模型对每个 Prompt 生成多个候选回答
        ↓
用 Reward Model / 人类 / 规则给回答打分
        ↓
筛出最高分回答
        ↓
把 Prompt + 最优回答 作为新 SFT 数据
        ↓
再训练模型
~~~

核心就是：

~~~text
生成 -> 筛选 -> 再 SFT
~~~

它的优点：

- 实现简单；
- 不需要 PPO；
- 不需要 DPO 损失；
- 训练稳定；
- 数据可解释，方便调试。

它的缺点：

- 上限不如 RL 方法；
- 只能学习自己已经生成过的高分回答；
- 探索能力有限；
- 多轮采样和筛选成本也不低。

拒绝采样常用于对齐训练的“热身阶段”。

比如：

~~~text
先用拒绝采样把模型推到一个不错的起点
再用 DPO / PPO / GRPO 做进一步偏好优化
~~~

一句话：

**拒绝采样就是让模型多生成几个答案，筛出最好的答案，再拿这些好答案继续做 SFT。它简单稳定，但探索能力不如真正的 RL。**

---

## 5. RLAIF：让强 AI 当老师

RLAIF 全称是 **Reinforcement Learning from AI Feedback**。

它可以理解为 RLHF 的变体。

RLHF 是：

~~~text
人类给反馈
~~~

RLAIF 是：

~~~text
强 AI 给反馈
~~~

也就是说，让一个更强的教师模型来评价候选回答，判断哪个更好、哪个更差，然后构造偏好数据。

流程是：

~~~text
当前模型生成多个回答
        ↓
强 AI 教师模型评价回答
        ↓
构造 chosen / rejected 偏好数据
        ↓
用于 Reward Model / DPO / 拒绝采样 / RLHF
~~~

RLAIF 主要解决的是 RLHF 的人工标注成本问题。

人工标注偏好数据非常贵，因为标注员需要：

- 阅读 Prompt；
- 阅读多个回答；
- 判断哪个更好；
- 保持标注一致性；
- 对复杂任务还需要专业知识。

RLAIF 的优点：

- 标注成本更低；
- 标注速度更快；
- 更容易规模化；
- 可以结合 Constitutional AI；
- 适合大规模构造偏好数据。

RLAIF 的缺点：

- 依赖一个更强的教师模型；
- 教师模型的偏见可能被学生继承；
- AI 偏好不一定等于真实人类偏好；
- 调用强模型本身也有成本。

一句话：

**RLAIF 是用强 AI 替代或辅助人类标注偏好数据，本质上是把反馈来源从人类扩展到了 AI。**

---

# 五种方法横向对比

| 方法 | 是否使用 RL | 需要的核心模型 | 训练稳定性 | 效果上限 | 典型特点 |
|---|---|---|---|---|---|
| RLHF / PPO | 是 | Policy、Reference、Reward、Value | 较差 | 高 | 经典但复杂 |
| DPO | 否 | Policy、Reference | 好 | 中高 | 简化 RLHF，开源常用 |
| GRPO | 是 | Policy、Reference、Reward 或规则奖励 | 较好 | 高 | 砍掉 Value Model，适合推理任务 |
| 拒绝采样 | 否 | Policy、Reward 或筛选器 | 极好 | 中 | 生成、筛选、再 SFT |
| RLAIF | 可结合 RL | 强 AI 教师 + 后续训练模型 | 取决于后续方法 | 高 | 用 AI 替代人类反馈 |

---

# 它们之间到底是什么关系？

这几个方法不是简单的并列替代关系。

更准确的理解是：

~~~text
Post-Training 是大类
├── RLHF：经典人类反馈强化学习
├── DPO：把偏好优化变成监督学习
├── GRPO：改进 PPO，省掉 Value Model
├── 拒绝采样：采样、筛选、再 SFT
└── RLAIF：把反馈来源从人类换成 AI
~~~

其中：

- **RLHF / DPO / GRPO / 拒绝采样** 更像是训练方法；
- **RLAIF** 更像是反馈数据的来源；
- **拒绝采样** 经常作为热身或数据构造步骤；
- **DPO** 适合低成本偏好对齐；
- **GRPO** 适合可验证推理任务；
- **RLHF** 适合资源充足、追求高上限的场景。

# 实际工程中怎么组合？

## 1. Llama 2-Chat 路线

Llama 2-Chat 的公开对齐流程可以概括为：

~~~text
SFT
  ↓
拒绝采样
  ↓
PPO / RLHF
  ↓
最终 Chat 模型
~~~

重点注意：

**Llama 2-Chat 不是 DPO 代表。**

它的关键路线是：

~~~text
SFT + 拒绝采样 + PPO/RLHF
~~~

不要在面试里说成：

~~~text
SFT + 拒绝采样 + DPO
~~~

---

## 2. DeepSeek R1 路线

DeepSeek R1 的路线可以概括为：

~~~text
SFT 冷启动
  ↓
GRPO 第一轮
  ↓
拒绝采样筛选高质量推理链
  ↓
第二轮 SFT
  ↓
GRPO 第二轮
  ↓
最终 DeepSeek R1
~~~

它的核心特点是：

- 用少量高质量推理数据做冷启动；
- 用 GRPO 在数学、代码等可验证任务上做 RL；
- 用拒绝采样筛出高质量推理链；
- 再通过 SFT 学习更好的推理表达；
- 最后继续 GRPO 优化。

一句话：

**DeepSeek R1 不是单一方法，而是 SFT + GRPO + 拒绝采样的多轮组合。**

# 如何选型？

## 1. 资源有限，想快速做偏好对齐

优先选：

~~~text
DPO
~~~

原因：

- 实现简单；
- 只需要偏好对；
- 不需要训练 Reward Model；
- 不需要 PPO；
- 显存占用较低；
- 训练稳定。

适合：

- 开源模型微调；
- 业务小团队；
- 算力有限；
- 已经有 chosen / rejected 数据。

---

## 2. 数学、代码、推理任务

优先考虑：

~~~text
GRPO
~~~

原因：

- 这类任务答案可验证；
- 奖励可以由规则或测试器给出；
- 不一定需要人工偏好；
- 保留 RL 的探索能力；
- 适合提升复杂推理能力。

适合：

- 数学推理；
- 代码生成；
- 算法题；
- 可自动判分任务；
- Reasoning Model 训练。

---

## 3. 想先构造一个稳定基线

可以先用：

~~~text
拒绝采样
~~~

原因：

- 简单稳定；
- 不需要复杂 RL；
- 可以筛出一批高质量回答；
- 适合作为 DPO / RLHF / GRPO 前的热身。

---

## 4. 人工标注太贵，想大规模造偏好数据

可以用：

~~~text
RLAIF
~~~

原因：

- 用强 AI 教师模型批量标注；
- 降低人工成本；
- 可以生成 chosen / rejected；
- 可以接 DPO、RLHF、拒绝采样等方法。

---

## 5. 大厂资源充足，追求最高上限

可以考虑：

~~~text
完整 RLHF 或 GRPO 流程
~~~

原因：

- RL 方法有探索能力；
- 效果上限更高；
- 适合大规模对齐训练；
- 但工程复杂度和成本都更高。

# 关键概念补充

## 1. Reward Model 是什么？

Reward Model 是奖励模型。

它的输入是：

~~~text
Prompt + Answer
~~~

输出是：

~~~text
一个奖励分数
~~~

它的训练数据来自人类或 AI 的偏好排序。

例如：

~~~text
回答 A 比回答 B 好
~~~

Reward Model 要学会：

~~~text
score(A) > score(B)
~~~

训练好之后，它就可以代替人类批量给回答打分。

---

## 2. Reference Model 是什么？

Reference Model 是参考模型，通常是冻结的 SFT 模型。

它的作用是：

**约束当前训练的 Policy Model 不要离原来的 SFT 模型太远。**

为什么需要这个约束？

因为如果只追求 Reward Model 的高分，模型可能学会钻漏洞。

比如：

- 输出奇怪格式骗奖励模型；
- 过度迎合奖励模型偏好；
- 牺牲通用能力；
- 出现 reward hacking。

所以需要 KL 约束：

~~~text
当前模型不要偏离参考模型太远
~~~

---

## 3. Value Model 是什么？

Value Model 是 PPO 里用来估计“预期奖励”的模型。

PPO 需要优势函数：

~~~text
优势 = 实际奖励 - 预期奖励
~~~

Value Model 就是用来估计这个“预期奖励”。

但是 Value Model 很贵：

- 需要单独训练；
- 规模通常接近主模型；
- 占显存；
- 增加复杂度。

GRPO 的创新就是：

**不用 Value Model，而是用组内相对奖励估计优势。**

---

## 4. Reward Hacking 是什么？

Reward Hacking 指的是：

**模型学会了骗奖励模型，而不是真的变好。**

比如 Reward Model 偏好长回答，模型可能就疯狂输出很长但没用的内容。

或者 Reward Model 偏好礼貌表达，模型可能变得非常客套但不解决问题。

所以 RLHF 里需要：

- Reference Model；
- KL 约束；
- 人工评估；
- 安全评估；
- 多轮数据修正。

# 面试官可能追问

## 1. 为什么 SFT 之后还需要 Post-Training？

因为 SFT 只能让模型学会“按指令回答”，但不能保证回答是最优的、安全的、符合人类偏好的。

SFT 学的是：

~~~text
给定指令，模仿标准答案
~~~

Post-Training 学的是：

~~~text
多个答案里，哪个更好、更安全、更符合偏好
~~~

---

## 2. DPO 和 RLHF 是什么关系？

DPO 可以理解为 RLHF 的简化版。

RLHF 是：

~~~text
偏好数据 -> Reward Model -> PPO 强化学习
~~~

DPO 是：

~~~text
偏好数据 -> 直接训练模型
~~~

DPO 绕过了 Reward Model 和 PPO，把偏好优化变成了监督学习问题。

---

## 3. GRPO 和 PPO 最大区别是什么？

最大区别是：

~~~text
PPO 需要 Value Model
GRPO 不需要 Value Model
~~~

GRPO 用同一个 Prompt 下多个回答的组内相对分数来估计优势函数。

所以 GRPO 更省显存、更简单，也特别适合数学、代码这类可以自动判分的任务。

---

## 4. 拒绝采样是不是强化学习？

不是。

拒绝采样没有 PPO、没有策略梯度，也没有 RL loss。

它只是：

~~~text
生成多个回答
筛出最好的
再做 SFT
~~~

所以它本质上是迭代式监督微调。

---

## 5. RLAIF 和 RLHF 最大区别是什么？

区别在反馈来源。

~~~text
RLHF：人类给反馈
RLAIF：AI 给反馈
~~~

RLAIF 用强 AI 教师模型替代或辅助人类标注，降低偏好数据构造成本。

---

## 6. 这五类方法是不是互相替代？

不是。

它们经常组合使用。

比如：

~~~text
Llama 2-Chat：SFT + 拒绝采样 + PPO/RLHF
DeepSeek R1：SFT 冷启动 + GRPO + 拒绝采样 + 再 SFT + 再 GRPO
~~~

工程里通常不是只选一个方法，而是根据阶段和资源组合使用。

---

# 记忆版总结

Post-Training 是 SFT 之后继续提升模型质量的一整套方法。

SFT 解决的是：

~~~text
模型会不会按指令回答
~~~

Post-Training 解决的是：

~~~text
模型回答得好不好
模型安不安全
模型是否符合人类偏好
模型推理能力能不能继续提升
~~~

五类方法可以这样记：

1. **RLHF**
   - 人类反馈；
   - 训练 Reward Model；
   - 用 PPO 优化；
   - 经典但复杂。

2. **DPO**
   - 直接用 chosen / rejected 偏好对训练；
   - 不需要 Reward Model；
   - 不需要 PPO；
   - 简单稳定。

3. **GRPO**
   - PPO 改进版；
   - 砍掉 Value Model；
   - 用组内相对优势；
   - 适合数学、代码、推理任务。

4. **拒绝采样**
   - 生成多个答案；
   - 筛出最优答案；
   - 再做 SFT；
   - 简单稳定但上限有限。

5. **RLAIF**
   - 强 AI 当老师；
   - 替代或辅助人类标注；
   - 降低偏好数据成本；
   - 可以接 DPO / RLHF / 拒绝采样。

最终一句话：

**Post-Training 不是某一个单独算法，而是 SFT 之后的一整套对齐和能力增强方法。RLHF 是经典路线，DPO 是低成本简化路线，GRPO 是适合推理任务的新 RL 路线，拒绝采样是稳定的数据筛选再训练路线，RLAIF 是用强 AI 扩大偏好反馈来源的路线。真正工程里，它们通常组合使用，而不是互相替代。**
