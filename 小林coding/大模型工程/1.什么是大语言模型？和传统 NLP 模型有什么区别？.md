
## 一、面试标准回答

面试官问：**什么是大语言模型？它和传统 NLP 模型有什么区别？**

可以这样回答：

大语言模型本质上是一个用海量语料预训练、参数规模很大、通过自回归方式预测下一个 token 的生成式模型。

它和传统 NLP 模型最核心的区别有三点。

第一，**任务方式不同**。  
传统 NLP 通常是“一任务一模型”，比如分词、命名实体识别、情感分析、问答系统都要分别训练模型，最后通过 pipeline 串起来。而 LLM 是“一个模型干很多任务”，因为它把任务统一成了 `Prompt + 续写` 的形式。

第二，**输出范式不同**。  
传统 NLP 多数是判别式模型，输入文本后输出标签、类别或概率，比如判断情感是正面还是负面；LLM 是生成式模型，输入一段文本后继续生成文本，翻译、总结、问答、写代码都可以用同一个生成接口完成。

第三，**能力来源不同**。  
传统 NLP 更依赖显式监督训练，也就是“喂什么任务数据，就学什么任务”；而 LLM 通过大规模预训练，在模型规模、数据规模和算力放大后，会出现一些工程上可观察到的能力跃迁，比如上下文学习、多步推理、跨语言迁移等。

所以一句话总结：

> **传统 NLP 是多个专用模型组成的 pipeline，LLM 是用生成式预训练把各种语言任务统一到一个模型和一个接口里。**

---

## 二、这篇文章的核心观点

这篇文章主要讲的是：

> **LLM 的本质不是“参数多、会聊天”，而是用预测下一个 token 的统一训练目标，把大量 NLP 任务收编成了一个生成式模型。**

传统 NLP 的工作方式是：

~~~text
分词
  ↓
词性标注
  ↓
命名实体识别
  ↓
意图分类
  ↓
知识库匹配
~~~

每一步都需要一个专门模型，每个模型又需要自己的训练数据和标注体系。

LLM 的工作方式是：

~~~text
用户任务
  ↓
Prompt 表达任务
  ↓
模型继续生成文本
~~~

也就是：

> **用 Prompt 把不同任务变成同一种续写问题。**

---

## 三、传统 NLP 的问题

传统 NLP 通常是 pipeline 架构。

例如智能客服系统可能要经过：

~~~text
分词
  ↓
词性标注
  ↓
命名实体识别
  ↓
意图分类
  ↓
检索标准答案
~~~

这种方式的问题是：

- 每个任务都要单独训练模型；
- 每个模型都依赖人工标注数据；
- 前面步骤出错会传递到后面；
- 换一个领域通常要重新标注和训练；
- 整体系统复杂，维护成本高。

可以记一句话：

> **传统 NLP 是把复杂语言问题拆成很多小任务，再用多个专用模型拼起来。**

---

## 四、BERT 解决了一半问题

BERT 带来了“预训练 + 微调”的范式。

它的贡献是：

> **让模型先在海量无标注文本上学习通用语言表示，再在下游任务上微调。**

BERT 解决了传统 NLP 中“特征不通用”的问题。

也就是说，过去每个任务都要从头训练特征，现在可以共享一个预训练模型。

但 BERT 仍然没解决两个问题：

### 1. 任务还是不统一

BERT 输出的是向量表示，不是直接输出最终文本。

所以不同任务还要接不同的任务头：

~~~text
分类任务：接分类头
NER 任务：接序列标注头
抽取式问答：接 QA 头
~~~

每个任务仍然需要单独微调。

---

### 2. 生成能力不强

BERT 的预训练目标是 MLM，也就是遮住一部分词让模型猜。

它更擅长理解，不擅长连续生成长文本。

所以 BERT 更像是：

> **通用语言理解底座，但不是统一生成接口。**

---

## 五、LLM 的本质：预测下一个 token

LLM 最关键的转变是：

> **把所有 NLP 任务统一成预测下一个 token。**

主流 GPT、Claude、Qwen、DeepSeek 等模型，本质上都是自回归语言模型。

训练方式是：

~~~text
看到前面的 token
    ↓
预测下一个 token
    ↓
再把预测结果接回上下文
    ↓
继续预测下一个 token
~~~

例如：

~~~text
输入：我喜欢吃
输出：苹果
~~~

看起来只是续写，但它可以覆盖很多任务。

---

## 六、为什么预测下一个 token 可以做很多任务？

因为很多 NLP 任务都可以写成 Prompt。

例如：

~~~text
翻译：
把下面这句话翻译成英文：我喜欢你 ->
模型续写：I like you

情感分类：
下面这条评论是正面还是负面？这家店太黑了。答：
模型续写：负面

总结：
请用一句话总结下面内容：xxx。总结：
模型续写：...

代码生成：
写一个 Python 函数计算斐波那契数列：
模型续写：def fibonacci...
~~~

所以 LLM 的统一接口是：

~~~text
Prompt + 续写
~~~

这就是它能“一个模型干很多任务”的根本原因。

---

## 七、为什么 LLM 能学到这么多能力？

关键是三个因素一起放大：

~~~text
模型规模
训练数据
训练算力
~~~

CLM 训练不需要人工标注，因为互联网上的大量文本天然可以作为训练数据。

模型为了更好预测下一个 token，被迫学到：

- 语法；
- 事实知识；
- 语义关系；
- 推理模式；
- 编程逻辑；
- 多语言对应关系。

比如要预测：

~~~text
北京是中国的____
~~~

模型需要知道“北京是首都”。

要预测：

~~~text
如果 x = 2，那么 x² = ____
~~~

模型需要具备基本计算能力。

所以：

> **预测下一个 token 看起来简单，但在海量数据和大模型规模下，会逼迫模型学到复杂语言和世界知识。**

---

## 八、In-Context Learning：上下文学习

LLM 还有一个重要能力：

> **在 Prompt 里看几个例子，就能学会当前任务模式，而不需要更新参数。**

例如：

~~~text
苹果 -> apple
香蕉 -> banana
草莓 -> strawberry
橘子 ->
~~~

模型可以根据前面几个例子推断出：

~~~text
这是水果中文翻英文任务
~~~

然后输出：

~~~text
orange
~~~

这叫 **In-Context Learning**。

它的意义是：

> **很多任务不需要重新训练模型，只需要在 Prompt 里给例子。**

---

## 九、涌现能力

LLM 相比传统 NLP 一个重要现象是：

> **模型规模到一定程度后，会出现训练目标里没有显式教过的能力。**

常见例子包括：

- 多步推理；
- 上下文学习；
- 跨语言迁移；
- 复杂指令理解。

需要注意：

> **涌现能力在工程上能观察到能力跃迁，但学术上也有人认为部分涌现现象可能和评测指标有关。**

面试中可以稳妥地说：

> **随着模型规模、数据和算力放大，LLM 会表现出传统小模型不具备的能力跃迁。**

---

## 十、LLM 和传统 NLP 的三个本质区别

| 维度 | 传统 NLP | LLM |
|---|---|---|
| 任务方式 | 一任务一模型，pipeline 串联 | 一个模型干多任务，Prompt 统一接口 |
| 输出范式 | 判别式，输出标签 / 概率 | 生成式，输出文本 |
| 能力来源 | 显式监督训练，喂什么学什么 | 大规模预训练 + 涌现，可能学到没显式教过的能力 |

这三个区别可以分别理解为：

~~~text
工程层面：从多个小模型拼装，变成一个统一大模型
范式层面：从分类器思维，变成生成器思维
能力层面：从人工标注驱动，变成大规模预训练驱动
~~~

---

## 十一、这对工程团队意味着什么？

过去做 NLP 项目，核心流程通常是：

~~~text
确定任务
  ↓
标注数据
  ↓
训练专用模型
  ↓
部署多个模型
  ↓
维护 pipeline
~~~

现在做 LLM 项目，工程重点变成：

~~~text
Prompt 怎么写
    ↓
是否需要 RAG 补充外部知识
    ↓
是否需要微调
    ↓
如何评估生成质量
    ↓
如何控制成本和延迟
~~~

所以 LLM 不只是模型变大了，而是：

> **整个 NLP 工程范式都变了。**

## 十三、面试官可能继续追问

### 1. 为什么不能简单说 LLM 就是参数很多的模型？

因为参数多只是表象。

LLM 的关键在于：

~~~text
大规模预训练
自回归生成
预测下一个 token
Prompt 统一接口
多任务泛化能力
~~~

参数规模只是支撑这些能力的重要因素之一。

---

### 2. BERT 算不算 LLM？

BERT 是大规模预训练语言模型，但通常不把它当作现在意义上的 LLM。

原因是：

- BERT 主要是理解式模型；
- 输出的是 token 表示，不是直接生成长文本；
- 下游任务仍然需要任务头和微调；
- 它没有把任务统一成 Prompt + 续写接口。

所以 BERT 更像是 LLM 之前的过渡阶段。

---

### 3. 为什么 LLM 能一个模型做很多任务？

因为它把任务统一成：

~~~text
Prompt + 预测下一个 token
~~~

翻译、分类、总结、问答、代码生成都可以表达成续写问题。

---

### 4. LLM 和传统 NLP 的本质区别是什么？

可以从三层回答：

~~~text
任务方式：传统 NLP 一任务一模型，LLM 一个模型多任务
输出范式：传统 NLP 多为判别式，LLM 是生成式
能力来源：传统 NLP 依赖监督标注，LLM 依赖大规模预训练和能力涌现
~~~

---

### 5. 什么是 In-Context Learning？

In-Context Learning 是指：

> **模型不更新参数，只通过 Prompt 里的示例学习当前任务模式。**

例如给几个中英翻译例子，模型就能继续完成新的翻译。

---

## 十四、记忆版总结

这篇文章可以压缩成一句话：

> **LLM 的本质是用大规模自回归预训练，把各种 NLP 任务统一成 Prompt + 预测下一个 token 的生成问题。**

进一步记：

~~~text
传统 NLP：
一任务一模型
pipeline 串联
判别式输出
依赖人工标注
迁移成本高

BERT：
预训练 + 微调
解决了通用表示
但没有统一任务接口
生成能力较弱

LLM：
预测下一个 token
Prompt 统一接口
生成式输出
一个模型做多任务
大规模预训练带来上下文学习和能力跃迁
~~~

最重要的一句话：

> **传统 NLP 是“多个专用模型拼 pipeline”，LLM 是“一个生成式大模型用 Prompt 统一任务”。**
